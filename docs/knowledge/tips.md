# 調査Tips

## 検索ワードが見つからないとき

### 1. 成功事例に立ち返る

調査に煮詰まったら `docs/knowledge/principles.md` の成功事例を見直す。

```
質問: この成功事例は「どんな検索ワード」で見つかる不満から生まれた？

例: Typing Mind ($45k/月)
  → 「ChatGPT UI 使いにくい」「ChatGPT 不満」
  → 同じ構造で別サービスを探せる
```

### 2. 抽象度を変える

```
具体的すぎる: 「Notion 検索 AND OR できない」
  → 抽象化: 「Notion 検索 弱い」「Notion 代替」

抽象的すぎる: 「仕事 効率化」
  → 具体化: 「フリーランス 請求書 面倒」「エンジニア 見積もり 時間かかる」
```

### 3. 別の角度から探す

```
同じ不満を別の言い方で:
- 「使いにくい」→「分かりにくい」「複雑」「難しい」
- 「高い」→「安い代替」「無料」「コスパ悪い」
- 「遅い」→「重い」「時間かかる」「待たされる」
```

## 調査に煮詰まったとき

### 1. パターンを変える

パターンBで見つからなければ、パターンAやCを試す。

```
今やっていること: パターンB（既存サービスの隙間）
  → 切り替え: パターンA（SNSから日常の不便を探す）
  → 切り替え: パターンC（特定業界に絞る）
```

### 2. ターゲットを絞る

```
広すぎ: 「会計ソフト 不満」
  → 絞る: 「美容師 会計 面倒」「不動産営業 請求書」
```

### 3. 時間軸を変える

```
最新だけ見ている: 「2025」
  → 過去も見る: 過去の不満が今も解決されていないか確認
```

## 収益性を見極める3つの質問

調査中に常に自問する：

1. **継続利用するか？**
   - 1回きりの需要（利用規約生成など）は避ける
   - 毎日/毎週使うものを狙う

2. **収益/時間に直結するか？**
   - 「あったら便利」程度では払わない
   - 節約した時間 or 増えた売上 > 支払額

3. **緊急性があるか？**
   - 「いつか対応すればいい」は後回しにされる
   - 「今すぐ解決したい」痛みを探す

## 調査ログを残す

調査を実施したら、必ず `docs/research/logs/` にログを残す。

同じ調査を繰り返さないため、また学びを蓄積するため。

ファイル名: `YYYYMMDD_[調査テーマ].md`

記録する内容：
- 調査日時
- 調査パターン（A/B/C）
- 対象サービス or 業界
- 使用した検索クエリ
- 発見した不満（箇条書き）
- 競合状況
- 判定（有望/要検討/却下）と理由
