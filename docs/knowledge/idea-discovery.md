# アイデア発掘方法

## 重要な前提

**「こんなツールほしい」という声を直接探すのはNG**（既にある）。

成功パターン・成功事例は [principles.md](principles.md) を参照。

---

## パターンA: 自分の不便を探す

### 発掘方法

1. 過去1週間で「面倒だな」と思った作業を書き出す
2. それを解決するために何を使ったか？
3. その解決策に不満はないか？
4. 同じ不満を持つ人はどこにいるか？（検索、Reddit、Twitter）

### LLMの場合（「自分」がいない）

LLMには日常的な不便がないため、外部から「自分」を探す：

- **Blueskyデータから収集**: 困りごとクラスタを抽出
- **SNS検索**: 「〜が面倒」「〜できない」を検索
- **ユーザーにヒアリング**: 「あなたが日常で感じている不便は？」

---

## パターンB: 既存サービスの隙間を探す

### 発掘方法

1. 自分が毎日使うサービス/プラットフォームをリスト化
2. そのサービスの「レビュー」「不満」をReddit/Twitter/App Storeで検索
3. 「〜できない」「〜が欲しい」という声を収集
4. その声に対して、本体が対応しそうにない or 対応が遅そうなものを選ぶ

### 検索クエリテンプレート

```
# 基本パターン
"[サービス名] 不満 使いにくい [年]"
"[サービス名] 困っている できない [年]"
"[サービス名] 代替 乗り換え [年]"

# 深掘りパターン
"[サービス名] [具体的機能] 弱い 解決策 [年]"
"[サービス名] 高い 安い 代替 [年]"
```

## パターンC: 業界経験から探す

### 発掘方法

1. 自分の職歴・経験した業界をリスト化
2. その業界で「みんな困ってるけど我慢してること」を思い出す
3. なぜ既存ツールで解決されていないか分析
4. 業界特有の事情（規制、慣習、予算感）を考慮した解決策を考える

### LLMの場合（業界経験がない）

業界特化のコミュニティから情報収集：

- **Reddit**: r/[業界名], r/[職種名]
- **Twitter/X**: 業界ハッシュタグ
- **Slack/Discordコミュニティ**

検索クエリ：
```
"[業界名] 非効率 なんとかしたい"
"[職種名] 毎日 時間かかる"
"[業界名] Excel 管理 限界"
```

## アイデアの質を上げる（連想と深掘り）

**重要**: 表面的なニーズではなく、根本原因を探る

```
例: 「クライアントからの返信が遅い」

❌ 浅い発想: 返信催促ツール → 既にある、誰も使わない

✅ 深掘り:
  なぜ返信が遅い？
    → クライアントが忙しい
    → 何を返信すればいいかわからない
    → 優先度が低いと思われている

  根本解決は？
    → 返信しやすい形式で質問する（選択式など）
    → クライアント側の作業を減らす仕組み
    → 初期段階で期待値を合わせるオンボーディング
```

### 連想のフレームワーク

1. その不満を持つ人は「他に何に困っているか」？
2. その不満の「前後」には何があるか？（前工程・後工程）
3. その不満を「別の業界」で見たことはないか？
4. その不満を「テクノロジーなし」で解決するなら？

---

## 競合調査（必須）

発見した不満に対して：

1. **既存解決策を列挙** - 無料ツール、有料ツール、公式機能
2. **各競合の料金を確認** - 無料プランの有無
3. **競合の評判を確認** - 不満点があるか

### 判定基準

```
✅ 有望: 3基準すべて満たす + 競合が弱い
⚠️ 要検討: 2基準満たす + 差別化の余地あり
❌ 却下: 1基準以下 or 無料競合が強い
```

3基準（詳細は [principles.md](principles.md) 参照）：
- 継続利用するか
- 収益/時間に直結するか
- 緊急性があるか
