# 検証方法

**検証方法は複数ある。LP作成だけが正解ではない。**

---

## A. LP（ランディングページ）型

```
方法: LP作成 → 集客 → ウェイトリスト/事前決済
ツール: Carrd($19/年), Notion, HTML
成功基準: 20人以上が決済情報入力 or 有料予約
```

---

## B. 直接セールス型（LP不要）

```
方法: 見込み客に直接連絡 → 課題ヒアリング → その場で売る
事例:
- Postperk: メールだけで最初の売上（LP無し）
- Gymhopper: 15人の顧客をLP訪問前に獲得
- EditorNinja: 2顧客確定してからローンチ → 初日$898 MRR
成功基準: 5-10人が「作ったら買う」と約束（できれば前払い）
```

---

## C. コンテンツ先行型（バズ→誘導）

```
【TikTok/YouTube Shorts】
- 教育コンテンツ + さりげない製品デモ
- 21-34秒動画で280%コンバージョン向上

【Twitter/X Build in Public】
- 開発過程を毎日投稿（#buildinpublic）
- Float: ローンチ前に$12K獲得
```

---

## D. コミュニティ浸透型

```
【Reddit】90/10ルール必須
- 90%価値提供、10%だけ宣伝
- 2-4週間のアカウント育成が必要
- コンバージョン率がGoogle広告の4倍

【Slack/Discord】
- ニッチコミュニティに参加
- 課題ヒアリングから始める
```

---

## E. プラットフォーム活用型

```
【Product Hunt】
- 12:01 AM Pacific にローンチ
- 事例: Tally → 1日でユーザー倍増

【AppSumo】
- ゼロオーディエンスでも362顧客獲得の事例
- 検証目的なら有効（手数料注意）
```

---

## F. SEO長期型

```
- PMF確認後に投資（早期は時間の無駄）
- 事例: Learning Next → Udemyレビュー分析でSEO流入
```

```markdown
### 検証KPI
- 成功基準: [具体的数値]
- 失敗判定: [撤退条件]

```

```markdown
| 2-3 | [タスク] | [成果物] |
| 4-7 | [タスク] | [成果物] |

### 4. 検証KPI

- 価格: [検証時のフィードバック反映]
```
