# NeedScoop

**月10〜100万円の継続収益を生むサービスアイデアを発見・検証するためのナレッジベース**

---

## 目的

個人開発で成功するサービスアイデアを体系的に発見する。


---

## 成功パターン

| パターン | 概要 | 成功事例 |
|---------|------|---------|
| **A: 自分の不便** | 自分が困っていることを解決→同じ人に売る | Typing Mind ($45k/月) |
| **B: 既存サービスの隙間** | 大きなサービスの「痒いところ」を埋める | blocs.me ($4k売却) |
| **C: 業界経験** | 内部にいたから見えた課題を解決 | Prerender.io ($2.5M/年) |

詳細は [docs/knowledge/principles.md](docs/knowledge/principles.md) を参照。


### LLMでのアイデア出しにおける問題

LLMアシスタントには「自分」がいない：
- 日常的に感じる「小さな不便」がない
- 特定の業界での実務経験がない
- 毎日使っているサービスへの不満がない

つまり、パターンAやCのアイデア発掘の出発点となる「自分」が存在しない。

### アプローチ

「自分」＝不満や困りごとを持つペルソナを、外部から探す。

```
従来のアプローチ（失敗）:
「こんなツールほしい」を直接検索 → 既に誰かが作っている

NeedScoopのアプローチ:
生の不満・困りごとを収集 → 連想・推論 → サービスアイデア化 → 検証
```

## 使い方

### 1. サービスアイデアの調査を依頼

```
「サービスアイデアを探して」
「パターンBで〇〇の不満を調査して」
```

Claude Codeは `CLAUDE.md` のガイドに従い、ナレッジを参照しながら調査を進める。

### 2. 補助ツール（必要に応じて）

Web検索では得られない大量のSNSデータが必要な場合のみ使用。

```bash
# Blueskyからデータ収集
python scripts/collect.py --limit 1000

# 困りごとクラスタを抽出
python scripts/extract_ideas.py
```

## 重要な原則

1. **検証ファースト** - 開発前に売れることを確認する
2. **競合調査必須** - 無料ツールが既にあれば撤退
3. **2週間ルール** - 検証で反応なければ次へ
4. **3基準を満たすか** - 継続利用・収益直結・緊急性
